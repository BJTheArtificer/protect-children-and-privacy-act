# Manifesto for Private Adult Access on General-Purpose Computers

*This is the foundational document for the Arkansas Protects Our Children and Our Privacy Online Act. It was written by the lead author before any drafting collaboration began. It sets out the core architecture and the principles the bill is built to enforce.*

---

Computers should not become identity checkpoints.

If lawmakers are going to require age checks at the operating-system level, the system must be built to protect lawful adult privacy, preserve normal computer ownership, and avoid turning every device into a tool for surveillance.

This manifesto proposes a simple rule:

**prove adulthood once, unlock normal adult use, and reveal nothing more than necessary.**

## 1. The goal should be narrow

The purpose of an operating-system age system should be only to determine whether a person is an adult for purposes of creating or enabling unrestricted adult access on a device.

It should not become:

- a universal identity system
- a behavioral monitoring tool
- a permanent account-linking system
- a backdoor for governments or corporations to track lawful adult activity

The state has an interest in protecting children. It does not have a legitimate interest in forcing every adult to attach their identity to the ordinary use of a computer.

## 2. Age should be checked once, not over and over

An adult should be able to prove they are 18 or older one time through an authorized issuer.

Authorized issuers may include:

- computer retailers
- online computer services
- government offices
- other approved age-credential issuers

The issuer should check ID once, issue a one-time adult setup credential, and then leave the process entirely.

The issuer should function like a clerk checking ID for an adult purchase: verify, issue, and step away.

## 3. A one-time setup credential should unlock normal adult use

The credential should be used once during device setup, first-time ownership, or ownership transfer.

After it is redeemed, the operating system should create or authorize a normal unrestricted adult admin account.

That adult account should be functionally identical to the normal unrestricted admin accounts people already use today. It should not be a special surveillance mode or a limited "adult sandbox."

The system should not create a separate digital identity for the adult. It should simply unlock normal adult control of the machine.

## 4. Websites and apps should only get one answer

Websites and apps should never receive identity data, exact age, date of birth, or a reusable adult credential.

They should be allowed to ask only one narrow question:

```
is_18_plus
```

The operating system should answer only:

```
true or false
```

or `1` or `0`.

Nothing more.

No site or app should receive:

- legal name
- exact age
- date of birth
- government ID number
- stable account identifier
- stable device identifier
- reusable adult token
- persistent cross-site credential

## 5. The issuer must not stay in the loop

Once the setup credential is issued and redeemed, the issuer should not be contacted every time a person:

- opens a site
- installs software
- uses an app
- enters an adult community
- browses lawful adult material
- uses an unrestricted operating system

No company and no government office should become a central monitor of lawful adult access.

## 6. The setup credential must expire after use

The one-time setup credential should be:

- one-time use
- short-lived before redemption
- invalid after use
- purged after expiration or redemption

It may be delivered as:

- a scratch-off code
- an NFC tag
- a printed activation card
- a one-time digital setup token

Its job is not to become a permanent adult badge. Its only job is to authorize adult setup one time.

## 7. Used devices and ownership transfers must remain simple

People buy used computers, inherit devices, wipe drives, reinstall systems, and transfer ownership all the time.

Any lawful system must work for:

- new devices
- used devices
- refurbished devices
- ownership changes
- clean installs
- repair and recovery situations

A person taking ownership of a device should be able to obtain a new one-time adult setup credential and configure the machine as an unrestricted adult-owned system.

## 8. Professionals, students, and homelabbers need a practical long-term path

Some people create, wipe, reinstall, test, virtualize, or deploy systems constantly.

This includes:

- IT workers
- developers
- security professionals
- students in technical programs
- researchers
- system administrators
- homelabbers
- people managing home security, home networking, or automation systems

These users should be able to apply for a long-term adult access permit.

That permit should be easy to obtain. It should require only:

- a statement of intent
- a pledge not to abuse or transfer the credential
- agreement to report compromise
- agreement that a compromised key may be blacklisted
- annual renewal

The fee should be modest and ordinary, similar to a hunting license, homebrewer permit, or other common permit fee.

The permit should not require a burdensome licensing process, intrusive investigation, or ongoing surveillance. Its purpose is only to allow legitimate repeated adult system setup without forcing constant repurchase or reissuance of one-time credentials.

## 9. Compromise should be handled simply

If a key or permit credential is stolen, copied, leaked, or abused, there should be a simple process to:

- report compromise
- blacklist the compromised credential
- issue a replacement
- restore lawful adult access without forcing identity linkage into the operating system itself

Security should be handled by revoking bad credentials, not by turning every use into a tracked event.

## 10. Multiple issuers must exist

No single company should control adult access to computers.

There must be multiple authorized issuers so that:

- no monopoly forms
- people have alternatives
- systems stay interoperable
- no single company becomes the gatekeeper for lawful adult computing

## 11. This must not become a child-surveillance system by default

Any handling of minors should be optional, parent-initiated, and limited.

The state should not use operating-system age policy as an excuse to force every child into a default profiling, restriction, or behavioral-control regime.

The central issue is adult privacy and adult access to general-purpose computing.

## 12. The principle is simple

A computer is a general-purpose tool. Adults should be able to own and use one without placing their identity into a permanent, trackable operating-system registry.

If age checks are mandated, the least invasive workable model is:

- prove adulthood once
- issue a narrow credential
- use it once for setup
- unlock normal adult admin access
- let sites ask only `is_18_plus`
- return only true or false
- attach no identity to the answer

And for people whose work, education, or technical use requires repeated setup, provide a low-burden annual permit for long-term adult access credentials.

## Final statement

We reject the idea that child safety requires adult deanonymization.

We reject the idea that operating systems must become general identity checkpoints.

We reject the idea that lawful adult computer use should depend on constant surveillance, repeated callbacks to credential issuers, or persistent identity-linked access systems.

We support a narrow, practical, privacy-preserving model:

**verify once, unlock adult access, reveal only is_18_plus, and leave lawful adults alone.**
